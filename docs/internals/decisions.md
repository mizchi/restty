# Architecture Decisions

## Rendering Mode
- Default: grayscale atlas with hinting.
- Optional: LCD subpixel atlas (toggle).
- No SDF/MSDF path (removed; raster atlas only).

## Ligatures
- Enabled by default.
- Draw shaped glyphs across cell ranges and skip per-cell glyphs in that span.
- Cursor/selection rendered as overlays to avoid breaking ligatures.

## Fonts
- Bundle JetBrains Mono (OFL-1.1).
- Allow local fonts via queryLocalFonts (Chromium only, user gesture).

## WASM Strategy
- Custom Zig wrapper over ghostty-vt Zig API (`wasm/src/restty.zig`).
- Avoid relying on current C ABI (missing Terminal/RenderState).
- `reference/ghostty` tracks `mizchi/ghostty @ wasm-kitty`, a fork of
  `ghostty-org/ghostty`. The `main` branch of the fork is a plain mirror
  of upstream; all divergence lives on `wasm-kitty` as three shim commits
  that re-enable the Kitty graphics protocol on `wasm32-freestanding`:
  - Force `kitty_graphics = true` in `src/terminal/build_options.zig`.
  - `Timestamp` alias + `nowTimestamp`/`timestampOrder` helpers in
    `src/terminal/kitty/graphics_image.zig` (monotonic `u64` counter
    when `std.time.Instant` is unavailable).
  - `path_max_bytes` fallback for `std.fs.max_path_bytes`, comptime guard
    around `posix.realpath`, and `timestampOrder` threading through
    `src/terminal/kitty/graphics_storage.zig`.
- Earlier shim patches from the wiedymi era (`style.zig`, `mouse_shape.zig`,
  `quirks.zig`, stream compat) are no longer needed — upstream absorbed the
  `.lib` artifact, `LoadingImage.Limits`, and `sys.zig` injection points,
  so the only remaining gap is Kitty graphics + wasm timestamps.
- Sync workflow: `just ghostty-sync` → `just ghostty-verify` → `just
  ghostty-push`. Runbook: `docs/development/ghostty-sync.md`.
- Embed wasm binary into the JS bundle for browser use (no runtime fetch).

## Build Environment
- WASM compile requires Zig 0.15.2 targeting `wasm32-freestanding`
  (matches `reference/ghostty`'s `minimum_zig_version`).
- Native Zig 0.15.2 on macOS 26.4 cannot link its own build runner
  against libSystem (`__availability_version_check` et al.). `just
  ghostty-verify` works around this by running the compile inside an
  Apple container with Linux Zig downloaded on demand to
  `.cache/zig-linux/`. `container system start` must be running; `just
  ghostty-zig-linux` fetches the toolchain.
- Linux hosts with working native Zig can bypass the container:
  `cd wasm && zig build -Dtarget=wasm32-freestanding -Doptimize=ReleaseSafe`.
- `src/wasm/embedded.ts` is regenerated by `scripts/build-wasm.ts`, which
  currently calls native Zig and is therefore blocked on macOS 26.4. A
  follow-up will route it through the container path so tests can
  exercise fresh wasm. Until then, `embedded.ts` tracks the last
  successfully-built wasm from before the environment broke.

## Rendering Backend
- WebGPU primary; WebGL2 fallback.
- Keep shader and buffer layout compatible across backends.
